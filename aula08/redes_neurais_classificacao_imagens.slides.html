Introdução¶

  • Muitos problemas de classificao possuem mais do que uma classe. Por ex. ao classificar imagens de dgitos temos 10 classes (dgitos de 0 a 9), ou ao classificar imagens de produtos de moda (camisetas, camisas, calas, bolsas, etc.) tambm temos mais do que duas classes.

  • Poderamos pensar que, a sada da rede neural poderia ser dividia de forma equidistante para cada classe. Por exemplo, em um problema de 3 classes, poderiamos dividir a sada 0 a 1 em:

    • 0.0 para classe 1 (dog);
    • 0.5 para classe 2 (cat);
    • 1.0 para classe 3 (horse).

  • Essa abordagem no tras bons resultados, ainda mais medida que o nmero de classes aumenta.

  • Para tratar este problema necessrio utilizar uma outra funo de ativao na ltima camada: Softmax.

  • Para que Redes Neurais possam ser usadas em problemas de classificao de mltiplas classes necessrio criar um neurnio de sada para cada classe.

Softmax¶

Ao utilizar um neurnio de sada para cada classe teremos um nmero entre 0 e 1 para cada um deles. Poderamos pensar que esse valor j poderia ser a probabilidade de cada classe acontecer, mas vamos ver um exemplo:

Situao 1: neste exemplo de 3 classes, vamos supor que para uma determinada de dados a sada fosse $[0.9,0.1,0.2]$. Podemos pensar que esse $0.9$ corresponde probabilidade de ser da classe 1, no entanto a soma de todas as sadas $1.2$, portanto, no pode corresponder a um valor de probabilidade.

Situao 2: vamos supor que para uma outra entrada a sada $[0.7, 0.7, 0.2]$. Alm de a soma ser mais do que 1, temos as classes 1 e 2 em um empate com valor 0.7. A probabilidade de ambas deveria ser a mesma, mas no poderia ser 0.7, j que a soma de ambas daria um valor maior do que 1.

  • Para resolver esse problema existe a funo Softmax

  • Esta funo permite transformar as sadas da rede neural em valores de probabilidades. A funo matemtica que permite transformar o vetor dessa sada da rede em uma distribuio de probabilidades a seguinte:

$$ \text{softmax}(y_i) = \cfrac{e^{y_i}}{\sum_i{e^{y_i}}} $$

  • onde $y_i$ o valor de sada do neurnio da classe $i$.

  • No numerador temos a funo exponencial de $y_i$, que retorna algum valor positivo.

  • No denominador temos a soma das funes exponenciais dos valores de sada para todas as classes, que tambm ser um valor positivo.

  • Essa relao entre o numerador de denominador faz com que a soma de todas as sadas de probabilidade seja 1.

Podemos definir a nossa funo em Python para realizar esse clculo

In [3]:
import torch 
def softmax(x):
    return torch.exp(x)/torch.sum(torch.exp(x))

Rede Neural com Saída Softmax¶

Em termos de implementao em Pytorch a nossa rede ficaria da seguinte forma:

In [6]:
from torch import nn
import torch.nn.functional as F

class Network_Multiclass_1(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(in_features=4, out_features=5)        
        self.fc2 = nn.Linear(in_features=5, out_features=3)        
                
    def forward(self, x):
        x = torch.sigmoid( self.fc1(x) )        
        x = F.softmax( self.fc2(x) , dim=1)
        return x

Alm disso, tambm temos que mudar a funo para medir o erro, pois ele que orienta o aprendizado da rede e tem influncia no clculo do gradiente. Ao utilizarmos softmax como sada da nossa rede devemos utilizar como critrio medio do erro a funo CrossEntropyLoss. Quanto ao otimizador vamos manter Stochastic Gradient Descent (SGD).

Segue abaixo um exemplo de implementao:

Network Multiclass 2¶

Arquitetura da Rede Neural:

  • Input Layer: 784 neurnios correspondentes a cada pixel da imagem de dimenses (28x28=784)
  • Hidden Layer: 20 neurnios com funo de ativao sigmoidal $f_\sigma=\frac{1}{1 + e^{-x}}$
  • Output Layer: 10 neurnios com funo de ativao Softmax $f_\gamma =\frac{e^{y_i}}{\sum_i{e^{y_i}}}$

In [44]:
def train_epoch(model, trainLoader, optimizer, criterion):
    model.train()
    losses = []
    for images, labels in trainLoader:
        # (0) Prepare data
        (batch_size, _, width, height) = images.shape
        images.resize_(batch_size, width*height)
        
        optimizer.zero_grad()
        # (1) Forward
        y_pred = model(images)    
        # (2) Compute error
        loss = criterion(y_pred, labels)
        # (3) Compute gradients
        loss.backward()
        # (4) Update weights
        optimizer.step()        
        losses.append(loss.item())
        
    model.eval()
    return losses

Verificiando a Acurácia¶

def eval_model(model, loader):
    measures = []
    total = 0
    correct = 0
    for images, labels in loader:
        # (0) Prepare data
        (batch_size, _, width, height) = images.shape
        images.resize_(batch_size, width*height)
        
        # (1) Get the most probably class
        _, y_pred = torch.max(model(images),1)
        
        total += labels.size(0)
        correct += (y_pred == labels).sum().item()
        
    measures = {'acc' : correct/total}
    return measures

Análise dos Resultados¶

Ao analisarmos o grfico verificamos que em 10 iteraes o loss da rede neural diminuiu em apenas 0.6, de 2.3 para 1.7.

Isso acontece quando temos muitos neurnios com funo de ativao sigmoidal, pois a derivada da funo de ativao sigmoidal costuma dar valores pequenos para os gradientes. Este problema chamado de Dissipao de Gradiente.

Para exemplificar isso vamos construir um grfico que simule o valor de uma combinao linear para a funo de ativao sigmoidal de um neurnio. Considere intervalo xx de -3 at 3, o resultado da funo de ativao sigmoidal desses dados em yy e o resultado da derivada da funo sigmoidal em yy_d.

In [129]:
xx = torch.tensor( np.arange(-3, 3, 0.01) )
yy = torch.sigmoid(xx)
yy_d = torch.sigmoid(xx) * (1 - torch.sigmoid(xx))

Função de Ativação Sigmoidal vs sua Derivada¶

In [130]:
fig, (ax1,ax2) = plt.subplots(1,2,figsize=(12,4)); plt.ylim((0.05,1.05))
ax1.plot(xx.detach().numpy(), yy.detach().numpy())
ax1.set_title('Funo de ativao sigmoidal')
ax2.plot(xx.detach().numpy(), yy_d.detach().numpy())
ax2.set_title('Derivada da funo de ativao sigmoidal')
plt.show()

A derivada da funo retorna vamos menores para entradas mais prximas de -3 e 3 e valores maiores quando est prxima a 0. Isso indica que quando a entrada para a funo sigmoidal for prxima de 0 o gradiente vai ser alto para que os pesos sejam alterados com mais fora e na prxima iterao o valor de entrada seja mais negativo ou mais positivo.

Network Multiclass 5:¶

Arquitetura da Rede Neural:

  • Input Layer: 784 neurnios correspondentes a cada pixel da imagem de dimenses (28x28=784)
  • Hidden Layer 1: 40 neurnios com funo de ativao ReLU $f_\sigma(x)=ReLU(x)$
  • Hidden Layer 2: 20 neurnios com funo de ativao ReLU $f_\sigma(x)$
  • Output Layer: 10 neurnios com funo de ativao Softmax $f_\gamma =\frac{e^{y_i}}{\sum_i{e^{y_i}}}$

Implementação da Rede Neural¶

class Network_Multiclass_5(nn.Module):
    def __init__(self):
        super().__init__()
        # From 784 to 40
        self.fc1 = nn.Linear(in_features=784, out_features=40)
        # From 40 to 20
        self.fc2 = nn.Linear(in_features=40, out_features=20)
        # From 20 to 10
        self.fc3 = nn.Linear(in_features=20, out_features=10)
        
    def forward(self, x):
        # O input passa pela primeira camada
        x = torch.relu( self.fc1(x) )
        # O input passa pela segunda camada
        x = torch.relu( self.fc2(x) )
        # O input passa pela terceira camada  
        x = F.softmax( self.fc3(x) , dim=1 )
        return x
